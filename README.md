# NAS工作流测试系统 - 启动说明

## 🚀 如何使用

### 方法一：直接打开（推荐）
1. 找到文件：`nas_test.html`
2. 右键点击文件
3. 选择「打开方式」
4. 选择任意浏览器（Chrome、Firefox、Edge等）

### 方法二：使用HTTP服务器（可选）
如果您希望通过HTTP服务器访问，可以在命令行中运行：

```bash
# 使用Python 3
python -m http.server 8080

# 或使用Python 2
python -m SimpleHTTPServer 8080

# 或使用Node.js
npx http-server -p 8080
```

然后在浏览器中访问：`http://localhost:8080/nas_test.html`

## 📋 系统功能

### 1. 游戏化关卡设计
- **共4个关卡**，覆盖从设计到归档的全流程
- **角色切换**：设计师、负责人/运营
- **实时反馈**：选择答案后立即显示正确/错误

### 2. 交互式界面
- **左侧训练面板**：
  - 场景描述
  - 操作选项
  - 关卡导航
  - 结果反馈

- **右侧监控面板**：
  - NAS实时状态
  - 文件夹结构展示
  - 文件位置动态更新
  - 操作日志记录

### 3. 关卡内容

| 关卡 | 标题 | 角色 | 场景 |
|------|------|------|------|
| 1 | 设计师上传 | 设计师 | 完成设计后上传文件 |
| 2 | 负责人评审(不通过) | 负责人/运营 | 发现问题并退回文件 |
| 3 | 负责人评审(通过) | 负责人/运营 | 确认通过并移动文件 |
| 4 | 资产归档 | 负责人/运营 | 将文件归档到正式资产库 |

## 🎮 使用指南

1. 阅读场景描述，了解当前角色和任务
2. 从选项中选择正确的操作流程
3. 观察右侧监控面板的变化
4. 根据反馈学习正确的操作方式
5. 使用「上一关」或「下一关」导航不同关卡

## 📁 项目文件

- `nas_test.html`：交互式测试系统主文件
- `nas_workflow.html`：原始静态展示页面
- `README.md`：详细的NAS工作流规范文档

## 💡 注意事项

- 页面支持响应式设计，可在不同设备上使用
- 所有操作都是模拟的，不会影响真实的NAS系统
- 建议按顺序完成所有关卡，以获得完整的学习体验
